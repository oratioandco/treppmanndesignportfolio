§ 03 · OPEN ISSUES & NEXT MOVES

v2 changes vs v1

  • Filled silhouettes → outlined / line-art via stroke-dasharray for the pixel-broken edge character
  • Three dash patterns in use: out-tight (3 1) for foreground, out-loose (2 2) for mid-ground, out-sparse (1 3) for distant buildings — gives each element its own texture, like Redaction's per-character distress
  • Reichstag dropped (was the weakest element)
  • TV Tower sphere redrawn as a discrete pixel circle (rect cluster forming a round shape)
  • Building rooflines now varied — stepped, slanted, flat — for skyline rhythm
  • Berliner Dom gets a sparse colonnade hint (vertical 1px ticks on the base)
  • Tree foliage = outlined circle with a few internal "leaf" dots, not a filled blob
  • Ground line is now dashed too — pixel-broken

If v2 is the direction

  • Add animation: stroke-dashoffset shifts subtly over time to give the outlines a "shimmer / alive" feel like the Redaction headline
  • Clouds drift slowly horizontally (60s loop)
  • Rain falls (~2s loop, multiple offsets)
  • Sun pulses gently (radius / opacity, 5s loop)
  • Pope's Revenge cross fades in over 1.5s when sunny state activates
  • Then wrap in BerlinFooter.astro + add Open-Meteo weather fetch + mount in Layout.astro

Still to decide

  • Is the line-art aesthetic right or should we go further (even sparser, even more broken-up)?
  • Should we add a NIGHT state (moon + a few stars) for post-sunset visits?
  • Should there be a small Departure Mono caption: e.g. BERLIN · ⛅ 12°C?
  • Tree: keep on the right, or move inward? (Currently still rightmost element)
  • Should the Pope's Revenge cross extend beyond the sphere a bit, since the real reflection does?
